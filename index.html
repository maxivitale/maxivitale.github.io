LangGraph Knowledge Agent

Full-stack system where a React 19 SPA streams every agent step in real time as a FastAPI backend runs a 10-node LangGraph workflow — compressing long conversation history, routing intent, decomposing queries, retrieving from ChromaDB and Memgraph in parallel, re-ranking with a cross-encoder, self-correcting on failures, and escalating to a Human-in-the-Loop when confidence is low. Documents enter through an async ARQ ingestion pipeline; progress surfaces via SSE.

Architecture at a Glance

Two independently deployable applications communicate via HTTP REST, WebSocket, and Server-Sent Events. The backend orchestrates all AI logic; the frontend provides a real-time, event-driven interface that mirrors every step of the agent workflow.

🧠

Agentic Chat Core

A 10-node LangGraph workflow with history summarization, self-correction loops, intent routing, query decomposition, and HITL escalation. State persists across turns via MemorySaver checkpointing.

📚

Hybrid RAG

Parallel retrieval over ChromaDB (dense vector chunks) and Memgraph (entity-enriched graph chunks). Combined pool re-ranked with a cross-encoder before reaching the generator.

⚡

Async Ingestion

Documents enqueued via ARQ (async Redis queue). Worker tokenises, chunks, embeds, and writes to both stores concurrently. Progress streams via Redis Pub/Sub → SSE.

🔒

Secure Real-time

WebSocket connections authenticate using single-use tickets (30 s TTL). The JWT never travels in the WebSocket frame. Every message is screened by a regex-based injection guard before the graph runs — direct and indirect prompt injection blocked in microseconds, no LLM call needed.

🔭

Observability

Full OpenTelemetry stack: traces → Tempo, metrics → Prometheus, logs → Loki. Trace IDs injected into every log record. Grafana dashboards auto-provisioned.

🛠️

MCP Server

Exposes the knowledge base over the Model Context Protocol at /mcp. Any MCP-compatible client can call hybrid_search, search_rag, search_graph directly.

The LangGraph Workflow

The heart of the system is a stateful, cyclical graph compiled with LangGraph. Each node is an async Python method. Conditional edges implement routing logic. The compiled graph persists per-session state with MemorySaver checkpointing.

nodes breakdown

Pre-graph: prompt injection guard

Before any LangGraph node executes, query_stream runs the raw user message through _check_injection() — a compiled regex (_INJECTION_PATTERNS) covering the most common injection vectors: override / forget / ignore instructions, jailbreak keywords, <system> XML tags, system code fences, and role-switching phrases. A match emits a single error event and returns — the graph is never invoked. For indirect injection via retrieved content, all KB and web chunks are wrapped in <retrieved_context> XML tags and the system prompt explicitly instructs the model to treat that block as data only. Zero latency cost for clean messages.

memory

summarize_history

Fires on every turn before classify_intent. No-op when estimated token count of history + summary is within HISTORY_TOKEN_THRESHOLD=6000. When exceeded, compresses older turns into a history_summary string via the small LLM, integrating any prior summary. _generate injects the summary as a SystemMessage and uses only the last KEEP_RECENT_TURNS=4 messages.

router

classify_intent

Structured LLM call (IntentOutput) routing to conversational (skip retrieval) or knowledge_query (full pipeline). Runs on every turn. Defaults to knowledge_query on failure.

decomposer

decompose_query

Detects multi-part queries, extracts up to 4 self-contained sub-queries (DecompositionOutput). A single sub-query is treated as no decomposition. Biased toward simplicity.

retrieval

hybrid_retrieve

Queries ChromaDB and Memgraph in parallel via asyncio.gather + thread-pool executor. When sub-queries present, runs one retrieval pair per sub-query (all parallel), then deduplicates. Cross-encoder re-ranks combined pool (top 5). Threshold: 0.3 cosine.

  • ChromaDB → dense vector chunks
  • Memgraph → entity-enriched chunks
  • Graph annotations stripped before scoring
fallback

web_supplement

Triggered only when KB returns zero docs. Fetches 5 DuckDuckGo results, filters by cosine similarity (threshold 0.2). Only relevant snippets reach the generator.

pre-check

evaluate_context

Pre-checks context relevance before the expensive generation call. If context is clearly off-topic, triggers exactly one preemptive rewrite+retrieve cycle (guarded by context_precheck_rewritten flag). Empty context caught without LLM call.

generator

generate

Streaming generation via ChatOpenAI. Injects context + retry hints ([retry:faithfulness], [retry:incomplete]) into the prompt. Passes conversation history for multi-turn coherence. Conversational replies bypass evaluate → END.

evaluator

evaluate

Structured LLM evaluation (EvaluationOutput) checks: language match, topical relevance, faithfulness (hallucinated domain facts), completeness (multi-part queries with clear omissions). Graph annotations stripped before evaluation.

retry

rewrite_query

Reformulates the query when retrieval failure is identified. Preserves the original language. Returns only the reformulated string. Routes back to hybrid_retrieve.

HITL

request_clarification

Invoked after MAX_ITERATIONS=3 without success. Uses LangGraph interrupt() to suspend the graph. Resumes via Command(resume=message) on the user's next message, resetting iteration=0 and re-entering from classify_intent.

# after retrieve → web fallback OR context pre-check
def _route_after_retrieve(self, state):
    total = len(state["rag_docs"]) + len(state["graph_docs"])
    return "web_supplement" if total == 0 else "evaluate_context"

# after context eval → rewrite OR generate
def _route_after_context_eval(self, state):
    # `is False` so None (not yet evaluated) falls through safely
    return ("rewrite_query"
            if state.get("context_relevant") is False
            else "generate")

# after evaluate → success / retry / escalate
def _route_after_evaluate(self, state):
    ev   = state.get("evaluation") or {}
    iter = state.get("iteration", 0)
    if ev.get("success", False):      return END
    if iter >= MAX_ITERATIONS:          return "request_clarification"
    if ev.get("failure_type") == "retrieval":
        return "rewrite_query"         # re-retrieve
    return "generate"                  # retry generation

Why is False and not not relevant?

context_relevant initialises to None. Using is False means an un-evaluated state safely falls through to generate rather than triggering an unintended rewrite. It's a small detail with a large blast radius if wrong.

Self-correction loop

The evaluate → rewrite → retrieve → generate cycle runs up to MAX_ITERATIONS=3 times. Retrieval failures rewrite the query; generation and faithfulness failures retry generation with targeted hints embedded in the prompt message.

Failure type → retry hint mapping

retrieval → [retry:context]  ·  generation → [retry:generation]  ·  faithfulness → [retry:faithfulness]  ·  incomplete → [retry:incomplete]

retrieval cross-encoder re-ranking
_rerank_docs() 
def _rerank_docs(self, query, rag_docs, graph_docs, top_k=5):
    all_docs = [("rag", d) for d in rag_docs] + \
               [("graph", d) for d in graph_docs]

    # strip graph annotations BEFORE scoring
    # (keep rich content for the LLM, fair scores for ranker)
    scoring = [_strip_graph_annotations(doc) for _, doc in all_docs]
    pairs   = [(query, text) for text in scoring]

    scores  = self._cross_encoder.predict(pairs).tolist()
    ranked  = sorted(zip(scores, all_docs),
                     key=lambda x: x[0], reverse=True)[:top_k]

    reranked_rag   = [doc for _, (src, doc) in ranked if src == "rag"]
    reranked_graph = [doc for _, (src, doc) in ranked if src == "graph"]
    return reranked_rag, reranked_graph

Two-stage retrieval pattern

The bi-encoder (sentence-transformers) retrieves cheaply at scale via ANN. The cross-encoder (ms-marco-MiniLM-L-6-v2) re-reads each (query, doc) pair jointly for much higher precision — but is too slow to run over the full corpus. Industry-standard pattern for production RAG.

Annotation stripping

Memgraph chunks include [Graph context: Entity A —rel→ B] annotations. Without stripping, annotation-dense chunks get inflated scores regardless of query relevance. Annotations are preserved in the final output passed to the LLM — only removed during scoring.

Thread-pool executor

The cross-encoder's predict() is synchronous. It runs in loop.run_in_executor(None, ...) to avoid blocking the async event loop — same pattern as similarity_search.

HITL Human-in-the-Loop escalation

interrupt() → suspend → resume

After 3 failed attempts, request_clarification calls LangGraph's interrupt(). The graph suspends — no more nodes execute. The backend emits clarification_needed via WebSocket. When the user responds, query_stream detects the session in _awaiting_clarification and invokes the graph with Command(resume=message), injecting the clarification as the interrupt result, resetting iteration=0, and continuing from classify_intent.

30-minute TTL guard

_awaiting_clarification maps session IDs to interrupt timestamps. A 30-minute TTL prevents a stale interrupted session from being accidentally resumed by an unrelated later message.

request_clarification + resume 
async def _request_clarification(self, state):
    # suspends graph — returns when user responds
    clarification = interrupt(
        "I tried multiple times but couldn't find"
        " a good answer. Could you rephrase?"
    )
    return {
        "query":       clarification,  # user's new message
        "iteration":   0,              # reset counter
        "evaluation":  None,
        "rag_docs":    [], "graph_docs": [],
        "context":     "",
        # all transient state cleared ↑
    }

# query_stream — resume or fresh start?
ts = self._awaiting_clarification.pop(session_id, 0.0)
if ts and (time.time() - ts) < _CLARIFICATION_TTL:
    graph_input = Command(resume=message)   # ← resume
else:
    graph_input = self._initial_state(message, session_id)

Async-first FastAPI application with dependency injection via @lru_cache provider functions. Clean separation between HTTP endpoints, services, and repositories.

ARQ + Redis Queue

The HTTP endpoint enqueues immediately and returns a task_id. The ARQ worker runs in a separate process and publishes granular progress events to a Redis Pub/Sub channel.

SSE Streaming

SseService subscribes to the Redis channel and yields events as an async generator consumed by EventSourceResponse. If the task already completed, the terminal status is returned immediately from a Redis key (1-hour TTL).

Dual-store concurrent ingestion

import_documents() runs ChromaDB and Memgraph ingestion concurrently via asyncio.gather. Token-based chunking (200 tokens, 30 overlap). Entity extraction uses Tenacity retry. Returns {documents_processed, entities_created}.

Shared EmbeddingService

A single SentenceTransformer (paraphrase-multilingual-MiniLM-L12-v2) shared across all services via DI. normalize_embeddings=True ensures dot product equals cosine similarity.

embedding_service.py

GraphRAG Document Store

Extends DocumentStoreInterface with a post-retrieval Cypher traversal (Document → MENTIONS → Entity → [r] → Entity) that appends related entities and relationships to each chunk's page_content.

Relevance score formula

ChromaDB returns L2 distances. Converted to similarity via 1 - L2²/2 and stored as metadata.relevance_score. Documents below 0.3 are filtered before cross-encoder re-ranking.

JWT + Redis Double Validation

Tokens are validated both cryptographically and against a Redis key (jwt:{username}:{jti}). Logout deletes the Redis key immediately — the JWT may still be mathematically valid but won't pass the store check.

auth_service.py

Why single-use WS tickets?

WebSocket upgrade URLs appear in server access logs and browser history. A long-lived JWT in the URL is a credential leak. Single-use tickets (30 s TTL) are fetched via an authenticated HTTP call and consumed atomically (GET+DEL pipeline), so even if the URL is logged, the ticket is already invalid.

OpenTelemetry Stack

setup_telemetry(app) wires up three providers during FastAPI lifespan. All are no-op when OTEL_ENABLED=False, adding zero overhead in development.

Traces → Tempo

OTLP exporter. Distributed traces across HTTP requests and LangGraph node executions.

Metrics → Prometheus

Scraped at /metrics. Latency, error rates, retrieval hit counts.

Logs → Loki

OTLP log exporter. trace_id + span_id in every log record for trace-log correlation.

Vendor-neutral signal routing

The OTel Collector aggregates all signals and fans out to Tempo, Prometheus, and Loki. Swapping Loki for Elasticsearch or Datadog requires only a collector config change — the application code is untouched.

Grafana auto-provisioning

Datasources are provisioned via config/grafana/provisioning/. The full observability stack starts with a single docker compose up.

Model Context Protocol Server

Any MCP-compatible AI client (Claude Desktop, Cursor, etc.) can query the knowledge base directly — without going through the chat WebSocket. The same vector search and graph enrichment that powers the chat is accessible as a first-class tool for external agents.

Key Patterns

🎫

Single-use WS tickets

Before each WebSocket connection, the frontend POSTs to /auth/ws-ticket for a 30 s single-use token. The JWT never appears in a WebSocket URL.

🔄

Callback switching

The ChatWebSocketClient singleton uses setCallbacks() to swap handlers between messages — one persistent connection serves multiple chat turns.

⏱

Sticky status (800 ms)

Significant states (evaluating, rewriting…) are guaranteed visible for ≥ 800 ms via a timestamp ref, preventing jarring flicker on fast backend paths.

📡

SSE via fetch

Ingestion progress uses fetch + ReadableStream instead of native EventSource — the only way to send an Authorization header with SSE.

⚡

Proactive clear

On rewrite_start, the frontend immediately clears partially-streamed text before the explicit content_clear event, returning to the full spinner state without a flash of stale content.

🎨

Dark design tokens

Custom Tailwind config: #000000 bg, #ececf1 text, #da7756 accent. animate-pending pulses AI messages while streaming.

Known Trade-offs

Conscious scope decisions with a defined upgrade path when requirements change.

componentcurrentlimitationupgrade path
State persistenceMemorySaver (in-process) State is lost on process restart; no shared state across replicas LangGraph's PostgreSQL checkpointer is a drop-in swap — same interface, externalized state, enables horizontal scaling
Vector storeChromaDB (single-node) No distributed mode, no built-in auth, limited horizontal scaling Qdrant via DocumentStoreInterface — native sharding, gRPC, built-in auth; swap is isolated to the repository layer
Web fallbackDuckDuckGo (unofficial) No official API, aggressive rate limiting, no SLA — not reliable under production load Tavily Search API (native LangChain integration) or Brave Search API — both offer official endpoints with predictable rate limits
Embedding modelIn-process SentenceTransformer Fixed RAM footprint regardless of traffic; doesn't scale independently from the API Dedicated inference service (FastEmbed server, Hugging Face Inference Endpoints, or Triton) — decouples embedding throughput from API scaling
Observability infraDocker Compose (single host) No high availability, no fault tolerance — entire stack tied to one machine Kubernetes-managed deployment or managed cloud equivalents (Grafana Cloud, Google Managed Prometheus)
